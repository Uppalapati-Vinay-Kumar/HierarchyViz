Data Preprocessing Guidelines

These guidelines should be followed to visualize the data hierarchically without any issues.

1. Meaningful Column Names

  • Column names should clearly describe the data they hold.
  • Avoid vague names like name â€“ instead, use Country_Name or Book_Title.
  • Ensure consistency in column names across datasets.
  • Use underscores (_) or PascalCase for multi-word column names to improve readability.

2. Handling Commas in Column Values

  • Commas should only be used in hierarchical data.
  • For example, a column containing USA, NJ, Glassboro is acceptable as it represents a hierarchy.
  • If a column contains non-hierarchical text (e.g., book titles), commas should be removed or enclosed properly.
Examples:
  • Incorrect: The Great Gatsby, a novel, written by F. Scott Fitzgerald
  • Correct: The Great Gatsby - A Novel by F. Scott Fitzgerald

3. Converting Numeric Columns for Hierarchical Data

  • If a numeric column is meant for hierarchy, treat it as a string.
  • Add a single quote (') before each numeric value to ensure correct interpretation.
Examples:
  • Incorrect: 101, 102, 103
  • Correct: '101, '102, '103
  • This ensures that the numeric column is interpreted as categorical data instead of continuous numbers.
